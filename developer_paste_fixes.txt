def delete_schedule_for_participant(obj, participant_code: str):
    """
    Historical compatibility shim.
    We no longer delete ScheduleItem rows because they are the canonical
    source used by export and may already contain collected data.
    """
    root = _root_subsession(obj)
    if not root:
        return

    count = len(ScheduleItem.filter(subsession=root, participant_code=participant_code))
    logger.warning(
        "delete_schedule_for_participant: SKIPPED deletion of %s schedule rows for participant=%s",
        count,
        participant_code,
    )

- participant completes all 80 rounds
- export still includes all schedule rows
- slot remains active+completed

- replacement takes freed slot
- replacement data does not overwrite original participant sentence data

======================================================================
4. Honest limitation
======================================================================

These paste-in changes remove the most dangerous production behaviors.
They do not fully solve cross-process race conditions in slot assignment.
A complete solution would still need transactional allocation or stronger uniqueness guarantees around active cohort slots and participant-round schedule rows.
